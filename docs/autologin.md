# Autologin endpoint

A single `GET` URL that logs in a configured user when the request supplies the
right token, then redirects.

## URL contract

`autologin_urlpatterns()` returns a list of one URL pattern when the package is
active and `autologin.enabled=True`. When inactive, it returns `[]` so the URL
genuinely doesn't exist (no defensive 404 view).

```python
from django_dev_helpers.urls import autologin_urlpatterns

urlpatterns = [
    *autologin_urlpatterns(),
    # ...
]
```

The URL pattern is built once when Django imports your URLconf. If you toggle
`enabled` after that, the URLconf is already cached — restart Django.

## Token

`DEV_HELPERS_AUTOLOGIN_TOKEN` must match the request's `?token=...` query
parameter exactly. Comparison is timing-safe (`hmac.compare_digest`).

The token is generated by the AppConfig at startup if the env var is absent
(`secrets.token_urlsafe(32)` — 256 bits) and is written to
`.dev_helpers_token` (mode 0600). When `django-run-site` is the orchestrator,
*it* generates the token and exports the env var; dev-helpers picks it up.

## Request flow

```text
GET /__autologin__/?token=<value>
  ↓
1. is_active()                          ← 404 if not
2. host allowlist check                 ← 404 if not localhost-y
3. token env-var check                  ← 404 if missing
4. timing-safe token compare            ← 404 on mismatch
5. User.objects.get(field=value)        ← 404 on miss / multiple
6. login(request, user, backend=…)
7. extra_cookies set on response
8. flash_message via django.contrib.messages
9. HttpResponseRedirect(redirect_to)
```

Only `GET` is accepted. POST/PUT/DELETE return `405 Method Not Allowed`.

## Host allowlist

Default allowed: `localhost`, `127.0.0.1`, `[::1]`. Add more via
`autologin.allowed_hosts`. Glob patterns work:

```python
DJANGO_DEV_HELPERS = {
    "autologin": {
        "allowed_hosts": ["192.168.1.42", "*.ngrok.io", "*.localhost"],
    },
}
```

Hostname matching uses `fnmatch.fnmatchcase` after lower-casing both sides.

## Custom user lookup

By default the view looks up `User.objects.get(username="admin")`. For custom
user models set `user_lookup_field` and `user_lookup_value`:

```python
DJANGO_DEV_HELPERS = {
    "autologin": {
        "user_lookup_field": "email",
        "user_lookup_value": "admin@example.com",
    },
}
```

When `user_lookup_value` is *not* set explicitly,
`DEV_HELPERS_AUTOLOGIN_USERNAME` (set by run-site) overrides the default
`"admin"`.

## Extra cookies

Useful for cookie-banner libraries: a single `extra_cookies` entry can dismiss
a consent banner so the agent never sees it.

```python
DJANGO_DEV_HELPERS = {
    "autologin": {
        "extra_cookies": [
            {
                "name": "cookielaw_accepted",
                "value": "1",
                "max_age": 31536000,
                "samesite": "Lax",
                "path": "/",
                "secure": False,
                "httponly": False,
            },
        ],
    },
}
```

`name` is an alias for `key`; both work.
